Notes on the Illumina clip commands


ILLUMINACLIP:fastaWithAdaptersEtc:seed mismatches:palindrome clip
threshold:simple clip threshold

The way how the Illumina clip works.

It selects the 16 nucleotide window within the adapter called seed and using sliding window approach moves it across the read sequence. The Each nucleotide is converted in the matrix with one row and four columns. This makes it 64 digit matrix that is compared against the 16 nucleotide fragments within the read. popcount operation that checks how many differences there are in matrixes is performed. In the example the cutoff is set for 2 changes

ILLUMINACLIP:TruSeq3-SE.fa:2:30:10

Once the fragment that satisfies the condition is found, the full alignment algorithm is performed for entire adapter. The mismatch score is calculated.

The full alignment score is calculated as follows. Each matching base increases the alignment score by 0.6, while each mismatch reduces the alignment score by Q/10 (where Q is phred quality at this nucleotide). By considering the quality of the base calls, mismatches caused by read errors have less impact. A perfect match of a 12 base sequence will score just over 7, while 25 bases are needed to score 15. As such we recommend values of between 7 - 15 as the threshold value for simple alignment mode.

In the example above the simple clip score is set to 10

ILLUMINACLIP:TruSeq3-SE.fa:2:30:10

If the settings are set to the SE mode the palindrome threshold is not applied because it requires the paired end mode (forward and reverse reads to be trimmed simultaneously

ILLUMINACLIP:TruSeq3-SE.fa:2:30:10

Sop in single read SE mode 30 setting is just being ignored
